## 一、项目概述

### 1.1 项目背景

MindInput 是一款基于 Android IME 的心理健康检测与干预系统，通过分析用户打字行为和文本内容，实现心理风险的早期预警和温和干预。

### 1.2 Sprint #1 目标

```
🎯 核心目标：完成可安装测试的 Alpha 版本

验收标准：
✅  MindInput 键盘可在 Android 8.0+ 设备正常安装
✅  QWERTY 键盘 + 数字键盘 + 符号键盘可用
✅  打字行为特征（WPM/退格率/停顿）正确采集
✅  L1 休息提醒在连续打字 30 分钟后触发
✅  L2 对话卡片在风险评分达到阈值时触发
✅  L4 危机弹窗在检测到危机词时触发（不可关闭）
✅  代码通过 ./gradlew assembleDebug 编译
```

### 1.3 技术栈

| 层级 | 技术 |
|------|------|
| 平台 | Android IME (InputMethodService) |
| 语言 | Kotlin 1.9 |
| 最低 SDK | API 26 (Android 8.0) |
| 目标 SDK | API 34 |
| 异步 | Kotlin Coroutines |
| 安全 | Android Keystore (AES-256-GCM) |
| 网络 | OkHttp + Retrofit |

## 七、Git 工作流

### 7.1 分支策略

```
main (受保护)
  └── develop
        ├── feature/F1-001-gradle-config
        ├── feature/F2-001-keyboard-layout
        ├── feature/F3-001-l1-reminder
        ├── feature/B1-001-circular-queue
        └── ...

命名规范：
feature/{Epic}-{TaskID}-{short-desc}

示例：
feature/F1-001-gradle-config
feature/F2-002-key-response
feature/B1-001-circular-queue
```

### 7.2 Commit 规范

```
{Epic}: {Action} {Description}

动作类型：
- Add: 新增功能
- Fix: 修复 bug
- Refactor: 重构
- Update: 更新内容
- Remove: 删除内容
- Docs: 文档更新
- Test: 测试相关
- Chore: 杂项

示例：
F2: Add QWERTY keyboard layout with 26 keys
B1: Implement circular queue for key timestamps
F3: Add L4 crisis intervention dialog
B2: Add crisis keyword detector with 15+ keywords
```

### 7.3 PR 流程

```
1. Feature 分支开发完成
2. 确保本地测试通过
3. 创建 PR 到 develop
   ├── 标题：{Epic}: {TaskName}
   └── 描述：包含测试结果、截图
4. 指定 Tech Lead + 1 人 Review
5. 所有 CI 检查通过
6. 2 人 Approve 后合并
```
