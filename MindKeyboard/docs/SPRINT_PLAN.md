## 六、Code Review 规范

### 6.1 PR 流程

```
1. Feature 分支开发完成
2. 创建 PR 到 develop
3. 指定 2 人 Review
4. 所有 Reviewer Approve
5. CI 所有检查通过
6. Tech Lead 合并
```

### 6.2 Review 检查清单

```
□ 功能逻辑正确
□ 命名规范统一
□ 异常处理完善
□ 无硬编码敏感信息
□ 单元测试覆盖新增代码
□ 文档已更新（如需）
```

### 6.3 Git 规范

```
分支命名：feature/{epic}-{task-id}-{short-desc}

示例：
feature/F2-002-key-response
feature/B1-001-circular-queue
feature/F3-005-l4-crisis-dialog

Commit 规范：{Epic}: {Action} {Description}

示例：
F2: Add QWERTY keyboard layout
B1: Implement circular queue for key timestamps
F3: Add L4 crisis intervention dialog
```
