## 三、构建 APK

### 3.1 Debug 构建（开发测试用）

```bash
# 进入项目目录
cd MindKeyboard

# 清理 + 构建
./gradlew clean assembleDebug

# APK 输出位置
# app/build/outputs/apk/debug/app-debug.apk
```

### 3.2 Release 构建（生产环境）

```bash
# 1. 生成签名密钥（首次）
keytool -genkey -v -keystore mindinput.jks \
  -keyalg RSA -keysize 2048 -validity 10000 \
  -alias mindinput

# 2. 配置签名（创建 signing.gradle.kts）
# 参考 3.2.1 小节

# 3. 构建 Release APK
./gradlew assembleRelease

# APK 输出位置
# app/build/outputs/apk/release/app-release.apk
```

### 3.2.1 签名配置示例

```kotlin
// app/build.gradle.kts 添加
android {
    ...
    signingConfigs {
        create("release") {
            storeFile = file("mindinput.jks")
            storePassword = "你的密钥库密码"
            keyAlias = "mindinput"
            keyPassword = "你的密钥密码"
        }
    }
    buildTypes {
        release {
            isSigningConfig = signingConfigs.getByName("release")
            isMinifyEnabled = true
            proguardFiles(
                getDefaultProguardFile("proguard-android-optimize.txt"),
                "proguard-rules.pro"
            )
        }
    }
}
```

### 3.3 构建输出

```
app/build/outputs/apk/
├── debug/
│   └── app-debug.apk          # ~15-20MB
└── release/
    └── app-release.apk         # ~10-15MB (开启混淆后)
```

## 四、安装到设备

### 4.1 通过 ADB 安装

```bash
# 连接设备（确保 USB 调试已开启）
adb devices
# List of devices attached
# xxxxxxxx device

# 安装 Debug APK
adb install -r app/build/outputs/apk/debug/app-debug.apk

# 如果安装失败，尝试卸载旧版本
adb uninstall com.mindinput
adb install app/build/outputs/apk/debug/app-debug.apk

# 安装完成确认
adb shell pm list packages | grep mindinput
# package:com.mindinput
```

### 4.2 通过 OTA 推送（企业内部）

```bash
# 1. 将 APK 上传到内网文件服务器
# 例如：https://internal.example.com/apk/mindinput-debug.apk

# 2. 设备端访问链接下载
# 或使用 MDM 工具推送
```

### 4.3 通过应用商店分发

```
Play Store 发布流程：
1. 创建开发者账号（$25 一次性费用）
2. 创建应用
3. 上传 APK/AAB
4. 填写应用信息

   - 描述：打字习惯分析 + 情绪关怀
   - 截图：键盘界面 + 干预界面
5. 隐私政策页面（必须）
6. 提交审核（通常 1-3 天）
```

## 五、启用输入法

### 5.1 用户端操作步骤

```
Step 1: 打开「设置」
         ▼
Step 2: 进入「系统」→「语言与输入法」
         ▼
Step 3: 点击「屏幕键盘」
         ▼
Step 4: 找到「MindInput 键盘」
         ▼
Step 5: 点击开关启用
         ▼
Step 6: 首次弹出「完全访问」提示
         ├─ 「允许」→ 启用云端同步功能
         └─ 「拒绝」→ 纯本地模式
         ▼
Step 7: 设置默认键盘（可选）
         - 进入「默认键盘」
         - 选择「MindInput 键盘」
```

### 5.2 管理员批量部署

通过 MDM（移动设备管理）推送：

```xml
<!-- MDM 配置示例 (AirWatch) -->
<managedProvisioning>
    <deviceConfiguration>
        <inputMethodSettings>
            <enabledInputMethods>
                <packageName>com.mindinput</packageName>
            </enabledInputMethods>
            <defaultInputMethod>com.mindinput/.MindInputService</defaultInputMethod>
        </inputMethodSettings>
    </deviceConfiguration>
</managedProvisioning>
```

### 5.3 权限说明

| 权限 | 用途 | 必需 |
|------|------|------|
| `INTERNET` | 云端同步（可选） | 否 |
| `ACCESS_NETWORK_STATE` | 检测网络状态 | 否 |
| `VIBRATE` | 按键震动反馈 | 否 |

**隐私说明**：
- 纯本地模式不需要网络权限
- 完全访问仅用于可选的云端同步
- 原始文本永不离开设备

## 六、使用指南

### 6.1 日常使用

```
1. 打开任意文本输入场景
   - 微信聊天
   - 备忘录
   - 搜索框
   - 任何 App 的输入框

2. 切换到 MindInput 键盘
   - 方法1：长按地球图标，选择「MindInput 键盘」
   - 方法2：点击键盘切换图标

3. 正常打字输入
   - 键盘自动采集打字行为
   - 无需额外操作

4. 观察干预提示（达到触发条件时）
   - L1：30分钟后顶部出现休息提醒
   - L2：情绪低落时出现对话卡片
   - L4：输入危机词时出现热线弹窗
```

### 6.2 L1 休息提醒

```
触发条件：连续打字 30 分钟以上

表现：
┌─────────────────────────────────────────────┐
│ 🌿 你已连续打字 40 分钟，休息一下？    [好的] [稍后] │
└─────────────────────────────────────────────┘

用户操作：
- 点击「好的」：关闭提醒
- 点击「稍后」：5 分钟后再提示
- 继续打字：提醒自动隐藏
```

### 6.3 L2 对话卡片

```
触发条件：风险评分达到 L2 阈值（55-75分）

表现：
┌─────────────────────────────────────────────┐
│  💙 听起来今天有点难熬                       │
│                                              │
│  要聊聊是什么事吗？                          │
│  或者我陪你做个深呼吸？                       │
│                                              │
│           [聊一聊]  [深呼吸]  [不用]         │
└─────────────────────────────────────────────┘

用户操作：
- 点击「聊一聊」：打开倾诉入口（Phase 2）
- 点击「深呼吸」：显示 3-4-3 呼吸练习
- 点击「不用」：关闭卡片
```

### 6.4 L4 危机热线

```
触发条件：
- 检测到危机关键词（不想活、没意思等）
- 或风险评分 > 90

表现：
┌─────────────────────────────────────────────┐
│                                              │
│              ⚠️ 你还好吗？                   │
│                                              │
│  我注意到你可能正在经历一些困难。            │
│  你现在安全吗？                              │
│                                              │
│                                              │
│          [我想聊聊]    [我没事]             │
│                                              │
└─────────────────────────────────────────────┘

注意：此弹窗不可关闭，用户必须选择才能消失

用户操作：
- 点击「我想聊聊」：拨打危机热线 400-161-9995
- 点击「我没事」：关闭弹窗（仍建议关注）
```

## 八、故障排查

### 8.1 常见问题

| 问题 | 可能原因 | 解决方案 |
|------|---------|---------|
| 键盘不显示 | 未启用输入法 | 设置 → 系统 → 语言与输入法 → 启用 |
| 无法切换键盘 | 未设置默认 | 长按地球图标选择 |
| L1 不出现 | 时间未到 30 分钟 | 确认连续打字时长 |
| L4 不触发 | 危机词未匹配 | 检查输入内容 |
| APK 安装失败 | 签名冲突 | 卸载旧版本再安装 |
| 云端同步失败 | 网络问题 | 检查网络 + 确认已开启同步 |

### 8.2 日志获取

```bash
# 查看 MindInput 相关日志
adb logcat | grep -E "MindInput|MindKeyboard|TypingBehavior"

# 完整日志
adb logcat > mindinput_log.txt

# 清除日志
adb logcat -c
```

### 8.3 远程诊断

## 九、安全与隐私

### 9.1 数据分级

| 级别 | 数据类型 | 存储位置 | 上报 |
|------|---------|---------|------|
| **敏感** | 原始文本、聊天内容 | ❌ 从不存储 | ❌ 从不上报 |
| **高敏感** | 打字内容片段 | 本地加密 | ❌ 从不上报 |
| **中敏感** | 打字速度、情感得分、认知扭曲类型 | 本地加密 | ✅ 匿名特征 |
| **低敏感** | 匿名用户 ID、时段、风险等级 | 云端 | ✅ 匿名统计 |

### 9.2 隐私保护措施

```
1. 端侧处理
   └── 所有文本分析在本地完成

2. 去标识化
   └── 用户 ID = SHA-256(设备ID + salt)

3. 精度降级
   └── 时间精确到小时，非分钟/秒

4. 传输加密
   └── HTTPS + TLS 1.3

5. 数据最小化
   └── 仅收集必要数据
```

### 9.3 合规清单

- [ ] 隐私政策已发布

- [ ] 数据删除机制（用户可请求删除）
- [ ] GDPR 合规（如面向欧盟用户）
